# 安全政策

## 支持版本

我们积极维护以下版本的安全更新：

| 版本 | 支持状态 | 说明 |
|------|---------|------|
| 2.0.x | ✅ 活跃支持 | 最新稳定版，接收所有安全更新 |
| 1.0.x | ✅ 维护支持 | 仅接收关键安全修复 |
| < 1.0 | ❌ 不再支持 | 请升级到受支持版本 |

## 报告漏洞

### 报告流程

1. **不要公开 issue** - 避免漏洞信息泄露
2. **发送邮件到** security@example.com
3. **包含详细信息**:
   - 漏洞类型和描述
   - 复现步骤
   - 影响范围
   - 建议的修复方案（如有）

### 响应时间

- **确认收到**: 48 小时内
- **初步评估**: 5 个工作日内
- **修复计划**: 根据严重程度制定
- **公开披露**: 修复后协调进行

### 安全研究指南

我们欢迎负责任的漏洞披露：

✅ **允许**:
- 测试公开部署的环境
- 报告潜在漏洞
- 提出安全改进建议

❌ **禁止**:
- 破坏服务可用性
- 访问或修改用户数据
- 公开未修复的漏洞

## Drupal 安全特别说明

### 模块安全

- 仅使用官方 Drupal 模块
- 定期更新模块到最新版本
- 审查第三方模块的安全性

### 配置安全

- 遵循最小权限原则
- 禁用不必要的功能
- 定期审计配置

## 安全最佳实践

### 开发者指南

- 定期更新依赖
- 使用安全的编码实践
- 进行代码审查
- 运行安全扫描

### 用户指南

- 及时升级到最新版本
- 审查第三方依赖
- 遵循最小权限原则
- 监控异常行为

## 安全更新

安全更新将通过以下渠道发布：
- GitHub Security Advisories
- Release Notes
- 邮件通知（订阅者）
